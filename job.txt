장흥다원

쇼핑몰: https://store.jangheungdawon.com/
AWS 서비스 구성: EC2 + ELB + RDMS + S3 + CloudFront
: EC2에 nginx + Wordpress + woocommerce 설치 및 주소검색 플러그인 제작하여 추가
: 아임포트 모듈을 사용하여 PG사 연결(구 LG U+ 현 토스페이먼츠 및 카카오페이)

선택 이유
: 무중단 업데이트를 위해, ELB에 SSL설정을 하여 EC2 인스턴스를 2개 연결
: 추가 기능을 플러그인으로 만드는 것이 wordpress 업데이트 영향이 가장 적다고 판단.
: 최소 관리만 필요하도록(사장은 웹서비스 관리를 이해하지 못함) RDMS 자동백업 및 아임포트를 사용

구성 완료후 운영인력이 없어 운영도 병행함


발효고 온도측정

AWS 서비스 구성: IoT Core + API Gateway + Lambda + DynamoDB + S3
: IoT Core를 통해 MQTT로 통신하고, REST로 Lambda 호출
: Node.js로 함수 작성. S3는 Vue.js 로 구현한 페이지 하나로 구성 

선택 이유
: 많이 쓰인다는 NoSQL DB, Vue.js 등을 적용해 보고 싶었음.
: ESP8266 NodeMCU 1.0 & LM35(sensor) 사용
: 20km 떨어진 곳에서 조회하기 위해 IoT Core 사용
: 검색기능은 필요없어 DynamoDB 사용
: javascript가 익숙하여 Lambda 함수는 Node.js를 사용



구직 희망사항

1. 30분 이내 출퇴근
: 2020년생 아이가 있기 때문에 가까울 수록 좋음

2. 2인 이상 일하는 팀
: 한 명 부재시 대행 가능
: 팀으로 일하는 경험과 소통 기술을 쌓고 싶음.
: 홀로 업무를 익히는 것보다 빠르게 업무를 익힘

3. 소통이 가능한 업무문화
: 의견을 묵살하지 않는 상사
: 서로 업무상 잘못한 점을 이야기해도 업무 불이익이 없는 문화
: 위와 같은 소통이 안되면 불필요한 사내 스트레스를 받음

4. 업무량(야근 등)을 반영한 급여
: 만족하는 급여는 장기근속 요인

5.트랜드는 아니어도 뒤쳐지지 않는 기술 스택을 적용
: 뒤쳐진 기술 스택은 시대를 따라가지 못해 구인이 어려워짐
: 변화가 없으면 기존 인력만 의지하게 되어 고인물이 됨


입사 및 퇴사

1. 신명문고
입사 결정
: 자전거로 20분 거리
: 정시퇴근, 공휴일 보장

퇴사 결정
: 사라져가는 업종이라 이직 불가능
: 신규 입사자 없음
: 임금 상승 거의 없음

배운점
: 사업을 할 때 금전관련 신용은 중요하다.
: 재고관리는 꼼꼼해야 한다.
: 퇴사를 모두가 축하하는 업종도 있다.


2. 핑거 
입사 결정
: SI는 근로환경은 열악하지만, 망할일은 없어 급여는 보장되고 인상도 잘된다하여 택함.
: IT기업은 사용하는 기술이 비슷하고 개인 수준차이만 있을 거라 생각함.

퇴사 결정
: 일상적인 야근과 하도급 구조로 연차 사용은 거의 불가능해 지쳐감.
: 근무지, 근무자는 물론 업무가 프로젝트마다 변경되어 안정감이 없음.
: 프리랜서에 의존하여 팀 업무를 경험할 수 없음.
- 관리자(PM)과 3개월 계약 프리랜서로 프로잭트 진행
- 프리랜서와 정규직의 업무나 관리방법에 차이가 없음(각자 할당량만 함).
- 협업 툴은 사용할 이유가 없어 사용해본 사람이 없음.
: 급여만 보고 택했는데, app개발에 투입된 입사동기들에 비해 급여인상이 제일 낮았음. 
- 3% 이상의 급여 인상은 이직이나 프리랜서를 해야만 가능했음

배운점
: 쉬지않고 일만 하다간 세상을 보는 시야가 좁아져, 사람이 바보가 됨.
: 기술을 익힌 뒤 사용하는 것보다, 업무에 필요한 기술중 좋은 것만 선택하여 익혀가는 것이 바람직하다.
- 공통js를 1여년간 만들다보니 javascript에 익숙해지고 좋아짐.
- 더글라스 크락포드의 JavaScript : The Good Parts 처럼, 좋은 부분이 많은 언어임.
: 인터넷을 차단한 환경에선 업무효율이 낮아진다.


3. TMD 교육그룹
입사 결정
: 직속 상사가 친구

퇴사 결정
: 종교를 앞세운 회사(매일 성경공부시간/기도시간이 있었음)가 부담스러움.
- 급여나 복지를 종교활동으로 대신하고 있음.
: 웹서비스는 외주개발이면 충분하다 생각하여 외주관리자를 원했기에 원하던 업무가 아님.

배운점
: 소통이 되는 상사가 있으면 내부 스트레스가 적다.


4. 알라딘커뮤니케이션 
입사 결정
: 버스로 15분 거리
: 정시 퇴근 보장

퇴사 결정
: 1년 이상 일하는 사람이 드물어, 매니저가 직원을 비품처럼 관리함 
: 한 자세로 일해야 하고, 오전/오후 각 15분 이내로만 쉴 수 있어 허리에 무리가 감
: 오전/오후 근무시간 조정에 기준이 없었고, 매니저만 토/일 휴일이 가능했음.

배운점
: 건강이 첫 번째다.
: 의외로 평일 휴일이 좋으나, 토/일 중 하루는 쉴 수 있어야 사회생활이 가능하다.


5. 장흥청태전영농조합법인 
입사 결정
: 차로 20분 거리
: 차(茶)계를 경험

퇴사 결정
: 코로나로 인한 매출 감소를 직원에게 얻어내려 함(직원탓 및 개인차량까지 업무에 동원요구)
: 영업이익은 대표가 가지고, 회사는 정부지원사업으로만 운영

배운점
: 정부지원사업의 허실을 경험
: 사람이 모이는 곳에 인재도 있음
: 사람이 모두 떠나는 회사는 이유가 있음.


6. 핀테크
입사 결정
: 지하철로 30분 거리
: 안정적인 연봉
: 함께 일해봤던 선임의 추천

퇴사 결정
: CTO 역할 및 개발본부장 역할의 인물의 돌연 결근 및 퇴사로 개발팀 붕괴
- 본부장 잠적을 대표 및 임원도 1주일 넘게 방관
- 남은 직원이 대신하면 된다는 입장이라 개발팀 전원 퇴사 결정

배운점
: 사내 불통이 지속되면, 급여가 높아도 창의적인 사람은 전부 퇴사한다.
: 참고 지내는 것은 한계가 있고,사람이 떠나는 곳은 대게 높은 분이 원인이다.
: 부족한 인력으로 해내는 것을 당연시하면 안된다.


IT 경험

이제서야, 쿠팡과 같은 기업과 SI는 별개의 업종이라는 것을 알게 되었습니다.
쿠팡 면접시 코드테스트에서 DB구조를 모방하여 해결한 이유를 물었을 때,
데이터는 DB가 다뤄야하고 시간내에 해결하기 위해 익숙한 방식을 택했다고 했었습니다.
SI방식에 젖은 사람이란 인식을 주어 탈락했다 생각합니다.

당시엔 기술이 부족했다 생각하여, TDD니 애자일이니 같은 트랜드를 공부해보기도 했지만,
어떤 기술이 최선인지 고민이 없었던 것이 문제였습니다.
최신 기술을 사용해도 지시대로만 사용하거나, 바꿀 여력이 없어서 그냥 하던대로 하는 곳은 고인물이 되고 맙니다. 

그동안 php, C++, C#, java, javascript 등을 사용해 보았고 그 중 javascript를 가장 오래 사용했습니다.
google script나 aws lambda등 여러 곳에서 javascript 문법으로 필요한 기능을 구현하기 쉬워 선호합니다.
물론 어떤 언어를 쓰든, 더글라스 크락포드의 글처럼 장점만 취해서 사용하면 된다 생각합니다.
